| 📄 RAG 流水线 | ❌ | 无 PDF 解析、无分块、无向量化、无 Milvus 集成 |
| ⚙️ AI 配置 | env | 藏在 `.env`，改参数需重启，无 UI |
| 🐳 Milvus | docker-compose | 已编排但未启动 |
| 📚 教材 PDF | **28 份** | 人教社数理化生全套教材 + 数学手册 132MB |

### 核心缺陷

1. **题目太少** — 214 个知识点仅 20 道题，每个知识点平均不到 0.1 道
2. **动画太少** — 仅 6 种动画类型覆盖 13 个节点，大多知识点是"文字墙"
3. **没有用户系统** — 任何人都能做题，但数据无法归属
4. **没有 RAG** — AI 导师只靠 contentJson 里的知识，查不到教材原文
5. **AI 配置硬编码** — 改模型/Key 要重启服务
6. **内容不够有趣** — 大多知识点只有"定义式"的内容，缺互动和趣味性

### 2.2 技术决策

| 决策 | 方案 | 理由 |
|------|------|------|
| **认证方案** | JWT (jsonwebtoken) + bcrypt 密码哈希 | 无状态、轻量、前后端分离友好 |
| **Auth 路由** | Node 端 `routes/auth.ts` | 与现有 Node 后端复用 Prisma，不改 Java |
| **AI 配置存储** | `config/ai-config.json` | ZDMusic 模式，管理员 UI 直接写文件，热加载 |
| **向量模型** | Ollama `nomic-embed-text` | 本地免费，768 维，中文不错 |
| **PDF 解析** | Python pdfplumber + langchain.text_splitter | 中文支持好，开源成熟 |
| **向量存储** | Milvus Standalone (Docker 已有编排) | 已就绪，启动即可 |
| **RAG 检索** | Node 端 `@zilliz/milvus2-sdk-node` | Node 直接调 Milvus REST，无中间进程 |
| **用户注册策略** | 仅管理员可创建账号 | 用户决策：开放注册→游客/admin 建用户 |
| **前端路由** | `/login` `/register` `/forgot-password` | 新增 |

#### 关键实现细节

```typescript
// JWT 载荷
interface JwtPayload {
  userId: number;
  username: string;
  role: 'admin' | 'user' | 'guest';
}

// 中间件
function authMiddleware(requiredRole?: 'admin' | 'user') {
  return (req, res, next) => {
    const token = req.headers.authorization?.replace('Bearer ', '');
    if (!token) {
      // 游客模式：req.user = { role: 'guest' }
      req.user = { role: 'guest' };
      if (requiredRole) return fail(res, 401, '请先登录');
      return next();
    }
    try {
      req.user = jwt.verify(token, JWT_SECRET);
      if (requiredRole === 'admin' && req.user.role !== 'admin') {
        return fail(res, 403, '权限不足');
      }
      next();
    } catch {
      return fail(res, 401, 'Token 已过期，请重新登录');
    }
  };
}
```

### 5.4 Node 端 RAG 检索

```typescript
// ragService.ts
import { MilvusClient } from '@zilliz/milvus2-sdk-node';

export async function searchRag(query: string, topK = 5, minScore = 0.65) {
  // 调 Ollama 获取 query 的 embedding
  const embedRes = await axios.post(`${ollamaUrl}/api/embed`, {
    model: 'nomic-embed-text', input: query
  });
  const queryVector = embedRes.data.embeddings[0];

  // 检索 Milvus
  const client = new MilvusClient({ address: 'localhost:19530' });
  const result = await client.search({
    collection_name: 'zd_rag',
    vector: queryVector,
    limit: topK,
    params: { nprobe: 16 },
  });

  // 过滤低分
  return result.results
    .filter(r => r.score >= minScore)
    .map(r => ({
      text: r.text,
      source: r.source,
      score: r.score,
    }));
}
```

### 5.5 AI 导师集成 RAG

在 `aiService.ts` 的 `ask()` 方法中：

```typescript
async function ask(req) {
  // 1. 构建知识库上下文（现有逻辑：contentJson）
  const knowledgeContext = req.nodeId ? await buildKnowledgeContext(req.nodeId) : '';

  // 2. RAG 检索
  let ragContext = '';
  if (getAiConfig().ragEnabled && req.question) {
    const ragResults = await searchRag(req.question, getAiConfig().ragTopK);
    ragContext = ragResults.map(r =>
      `[来自《${r.source}》] ${r.text}`
    ).join('\n\n');
  }

  // 3. 组装 prompt
  const prompt = `
【知识库内容】
${knowledgeContext}

${ragContext ? `【教材参考资料】\n${ragContext}` : ''}

【用户问题】
${req.question}
  `.trim();

  // 4. 调 LLM
  return callAi(prompt);
}
```

## 十一、注意事项 & 关键踩坑

### 11.1 数据迁移

- 现有 SQLite 中有 `ExamAnswer` 表，`userId` 目前可能是 `0` 或 `1`
- 加了认证后需要：把游客答题记录的 `userId=0` 迁移为 `guest` 用户
- 或者给游客分配一个固定 `userId=0` 的虚拟用户

### 11.2 安全

| 风险 | 应对 |
|------|------|
| 密码明文存储 | 立即用 bcrypt 哈希 |
| JWT 泄露 | 设置过期时间（建议 7 天），提供 refresh token |
| API Key 泄露 | AI 配置页面 Key 输入用 `type="password"`，前端请求返回掩码 |
| XSS | 所有用户输入做转义 |
| SQL 注入 | Prisma 已防止 |

### 11.3 Milvus

- Docker `dockerProject/Milvus-Attu/docker-compose.yml` 依赖 etcd + minio
- 首次启动较慢（约 30-60 秒），注意 health check 配置
- 需要确认 etcd + minio 也在同一 Docker 网络下

### 11.4 Ollama

- `nomic-embed-text` 模型约 274MB，下载需要几分钟
- 首次 embedding 需要编译（CPU 或 Metal GPU），macOS 推荐用 Metal
- 建议先测单个 PDF 的 ingest，再全量跑 28 个文件
- 预期 28 个文件 ~14,000 chunks，Ollama embedding 约 3-5 分钟

### 11.5 Prisma 与 SQLite

- SQLite 不适合高并发写入（PDF ingest 不应该写 SQLite）
- PDF 向量化只写 Milvus，元数据可存一个简单的 `ingest-manifest.json`
- 如果后续要存 PDF 处理日志到数据库，建议新增 `DocumentProcessLog` 表
