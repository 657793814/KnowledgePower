## 一、架构概览

### 1.1 三层题目前端生成链路

```
数据（knowledge_node.contentJson）
  ↓  parseContent()
  ↓  提取 knowledgePoints / examples / strategies / commonMistakes
  ↓
规则引擎（bulk-questions.ts）
  ↓  genChoiceFromKeyPoints
  ↓  genFillFromFormulas
  ↓  genJudgeFromKeyPoints
  ↓  genChoiceFromExamples
  ↓
数据库（exam_question 表）
  ↓  GET /random /generate /model-train
  ↓
用户作答 → 批改 → 错题本 → 统计
```

### 1.2 核心流程

```
                  +-------------------+
                  |  contentJson      |  ← 结构化知识点内容
                  |  (定义/要点/公式   |
                  |   例题/策略/易错点) |
                  +--------+----------+
                           |
               +-----------v-----------+
               |   题目生成引擎         |
               |   ──────────────────  |
               |   questions.ts        |  手工录入的通用题目
               |   model-questions.ts  |  手工录入的模型专题题目
               |   bulk-questions.ts   |  从 contentJson 自动提取生成
               +-----------+-----------+
                           |
               +-----------v-----------+
               |   exam_question 表     |
               |   ──────────────────  |
               |   字段: title/options  |
               |   answer/explanation   |
               |   nodeId/questionType  |
               +-----------+-----------+
                           |
               +-----------v-----------+
               |   对外接口             |
               |   ──────────────────  |
               |   GET /random          |  随机出题
               |   POST /generate       |  自动组卷
               |   GET /model-train     |  模型专题训练
               |   POST /submit         |  提交批改
               |   GET /wrong-book      |  错题本
               |   GET /stats           |  学习统计
               +-----------+-----------+
```

## 二、题目数据模型

### 2.1 数据库表（exam_question）

```prisma
model ExamQuestion {
  id            Int      @id @default(autoincrement())
  nodeId        String                    // 关联知识点，如 "MATH-05-016"
  subject       String                    // "math" / "physics" / "chemistry"
  domain        String                    // "几何" / "力学" / "有机化学"
  level         String                    // "初中" / "高中"
  questionType  String                    // "choice" / "fill" / "judge"
  difficulty    Int                       // 1-5
  title         String                    // 题干
  options       String?                   // 选项 (JSON)
  answer        String                    // 正确答案
  explanation   String                    // 解析
  tags          String?                   // 标签 (JSON 数组)
  status        Int        @default(1)   // 0=禁用 1=启用
  deleted       Int        @default(0)   // 软删除
  createdAt     DateTime   @default(now())
  updatedAt     DateTime   @updatedAt
}
```

### 2.2 题型定义

| 类型 | questionType | 选项格式 | 答案匹配 |
|------|-------------|---------|---------|
| 选择题 | `choice` | JSON 对象 `{"A":"内容","B":"内容"}` 或数组 `["A.xxx","B.xxx"]` | 正确答案字母 `"A"` |
| 填空题 | `fill` | `null` | 答案字符串 `"3.14"` |
| 判断题 | `judge` | `null` | `"正确"` 或 `"错误"` |

### 2.3 三个种题目来源的分布

| 来源 | 生成方式 | 题目数量 | 文件 |
|------|---------|:--------:|------|
| **通用题目** | 手工录入（每学科少量） | 48 道 | `questions.ts` |
| **模型专题** | 手工录入（每模型 3-10 题） | — | `model-questions.ts` |
| 数学模型专题 | 手工录入（38 个模型） | ~400 | `model-questions-math.ts` |
| 物理模型专题 | 手工录入 | ~50 | `model-questions-physics.ts` |
| 化学模型专题 | 手工录入 | ~50 | `model-questions-chemistry.ts` |
| **批量生成** | 从 contentJson 规则提取 | **2252 道** | `bulk-questions.ts` |

## 四、批量自动生成（bulk-question）

### 4.1 整体架构

```typescript
// scripts/bulk-questions.ts
for each 知识点节点:
  contentJson → parseContent()
  └─ 定义 + 要点 + 公式 + 例题 + 策略 + 易错点

  生成流程:
  1. genChoiceFromKeyPoints  (要点→选择题)
  2. genFillFromFormulas     (公式→填空题)
  3. genJudgeFromKeyPoints   (要点→判断题)
  4. genChoiceFromExamples   (例题→选择题)

  写入：
  findThenUpdateOrCreate(幂等模式)
```

### 4.2 parseContent — 结构化内容提取

从知识点的 `contentJson.sections[]` 中按 type 提取各类内容：

```typescript
function parseContent(node: any): ParsedContent {
  const sections = content?.sections || [];
  for (const s of sections) {
    if (s.type === 'definition') {
      result.definition = s.content;
      result.formulas = s.formulas || [];
    }
    if (s.type === 'keypoints') result.keyPoints = s.items || [];
    if (s.type === 'example') result.examples = s.items || [];
    if (s.type === 'strategy') result.strategy = s.items || [];
    if (s.type === 'common-mistakes') result.commonMistakes = s.items || [];
    if (s.type === 'exam-focus') result.examFocus = s.content || '';
  }
}
```

### 4.3 genChoiceFromKeyPoints — 要点生成选择题

```typescript
function genChoiceFromKeyPoints(node, parsed): ExamQuestion[] {
  const kps = parsed.keyPoints;
  // 对每个要点：
  for (const kp of kps) {
    // 从要点内容中提取数字作为正确选项
    const numMatch = kp.match(/\d+\.?\d*/);
    correctNum = numMatch[0];

    // 生成 3 个干扰项
    干扰项1 = correctNum + randomOffset(1-3)
    干扰项2 = correctNum * 2 或类似数字替换
    干扰项3 = "无法确定" 或 "以上都不对"

    // 构建选择题
    title = `关于"${node.title}"，下列说法正确的是？`
    options = {A: 要点, B: 干扰1, C: 干扰2, D: 干扰3}
  }
}
```

**问题：** 数字提取简单，只适合数值型要点。非数值要点（如定义类）跳过。

### 4.4 genFillFromFormulas — 公式生成填空题

```typescript
function genFillFromFormulas(node, parsed): ExamQuestion[] {
  for (const formula of parsed.formulas) {
    // 从 $...$ 包裹的 LaTeX 公式中提取一个数字作为填空
    const numMatch = formula.match(/(\d+)/);
    填空题 = formula.replace(numMatch[0], '____');
    answer = numMatch[0];
  }
}
```

**问题：** 只填一个数字，不处理多空/表达式的复杂填空。

### 4.5 genJudgeFromKeyPoints — 要点生成判断题

```typescript
function genJudgeFromKeyPoints(node, parsed): ExamQuestion[] {
  for (const kp of parsed.keyPoints) {
    // 35% 概率翻转 → 错误题
    const shouldFlip = Math.random() < 0.35;

    if (shouldFlip) {
      // 数字翻转（如 3 → 2 或 4）
      const numMatch = statement.match(/(\d+)/);
      if (numMatch && parseInt(numMatch[1]) > 1) {
        newNum = origNum + randomOffset(±1)
        title = statement.replace(origNum, newNum);
      }
    } else {
      // 直接原样作为正确题
      title = statement;
      answer = "正确";
    }
  }
}
```

**核心规则：**
- 65% 概率生成"正确"题（原样提取要点）
- 35% 概率生成"错误"题（只翻转关键数字，不直接加"不是"）
- 长度小于 5 或大于 80 的要点跳过

### 4.6 genChoiceFromExamples — 例题生成选择题

```typescript
function genChoiceFromExamples(node, parsed): ExamQuestion[] {
  for (const ex of parsed.examples) {
    // 从答案中提取数字
    const numMatch = correct.match(/(\d+(\.\d+)?)/);
    // 生成 4 个选项，其中 1 个是正确答案，3 个是干扰项
    options = {
      A: correct,          // 正确答案
      B: wrong1(numMatch), // 正确数字 ± 随机偏移
      C: wrong2(numMatch), // 正确数字 ÷ 2 或 × 2
      D: wrong3("None"),   // "无法确定" 或 "以上都不对"
    };
  }
}
```

**⚠️ 已知 Bug（已修复）：**
- `generateWrongAnswers` 将枚举题号 `(1)3.14` 中的 `(1)` 的 `1` 当成数字替换
- 防御：`/\(1\)/.test(correct) && correct.length > 15` → 跳过多段枚举答案

### 5.1 接口


### 5.2 实现逻辑

```typescript
async function autoGenerate(req) {
  // 1. 按条件筛选候选题目
  const where = { status: 1, deleted: 0 };
  if (req.domain) where.domain = req.domain;
  if (req.level) where.level = req.level;

  const all = await prisma.examQuestion.findMany({ where });

  // 2. Fisher-Yates 洗牌
  for (let i = all.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [all[i], all[j]] = [all[j], all[i]];
  }

  // 3. 取前 N 道
  const picked = all.slice(0, Math.min(count, all.length));

  // 4. 创建试卷记录
  const paper = await prisma.examPaper.create({
    data: {
      title: domain ? `${domain} · 练习` : '综合练习',
      questionIds: picked.map(q => q.id).join(','),
      questionCount: count,
      mode: req.mode,
      timeLimit: req.timeLimit || null,
    }
  });

  // 5. 返回试卷（不含答案，含选项/解析/知识点名）
  return { id, title, questions: questionsVO };
}
```

### 5.3 Fisher-Yates 洗牌算法

完全公平的随机排列，每次迭代在当前剩余元素中随机选一个交换：

```typescript
// 对于数组 a[0..n-1]
for i from n-1 down to 1:
  j = random(0, i)  // 含 i
  swap(a[i], a[j])
```

时间复杂度 O(n)，空间 O(1) in-place，等概率生成所有 n! 种排列。

### 5.4 试卷 VO 组装

```typescript
async function toQuestionVO(q: ExamQuestion) {
```

## 六、提交批改与错题本

### 6.1 批改逻辑

```
POST /exam/submit
{
  "paperId": 1,          // 可选
  "answers": [
    { "questionId": 123, "userAnswer": "B" },
    { "questionId": 456, "userAnswer": "3.14" }
  ]
}
```

```typescript
async function submitAnswers(userId, req) {
  const qIds = req.answers.map(a => a.questionId);
  const questions = await prisma.examQuestion.findMany({
    where: { id: { in: qIds } }
  });
  const qMap = new Map(questions.map(q => [q.id, q]));

  let correct = 0;
  const details = [];

  for (const a of req.answers) {
    const eq = qMap.get(a.questionId);
    if (!eq) continue;

    // 答案匹配（大小写无关、首尾去空格）
    const isCorrect =
      eq.answer.trim().toLowerCase()
      === a.userAnswer.trim().toLowerCase();

    if (isCorrect) correct++;

    // 记录答题
    await prisma.examAnswer.create({
      data: { userId, paperId, questionId, userAnswer, isCorrect, score: ... }
    });

    details.push({
      questionId,
      userAnswer,
      correctAnswer: eq.answer,   // ✅ 提交后才返回正确答案
      correct: isCorrect,
      explanation: eq.explanation,
    });

    // 错题自动加入错题本
    if (!isCorrect) {
      await addToWrongBook(userId, questionId);
    }
  }

  return {
    totalCount, correctCount, wrongCount,
    score: (correct / total) * 100,
    details
  };
}
```

### 6.2 答案匹配规则

```typescript
eq.answer.trim().toLowerCase() === a.userAnswer.trim().toLowerCase()
```

严格字符串匹配。当前限制：
- **选择题：** 学生选 `A`，正确答案是 `A` → 精确匹配 ✅
- **填空题：** 学生答 `3.14`，正确答案是 `3.14` → 精确匹配 ✅ 
- **判断题：** 学生选 `正确`，正确答案是 `正确` → 精确匹配 ✅
- **不支持：** 答案近义匹配（如 `$\\sqrt{2}$` 和 `1.414` 不等价）
- **不支持：** 多空填空解析（如 `20, 15` 必须完全匹配逗号格式）

```typescript
async function getStats(userId, subject?) {
  // 1. 查询所有答题记录
  const allAnswers = await prisma.examAnswer.findMany({
    where: { userId }
  });

  // 2. 基础统计
  const total = allAnswers.length;
  const correct = allAnswers.filter(a => a.isCorrect).length;
  const accuracyRate = total > 0 ? (correct / total) * 100 : 0;

  // 3. 错题本总数
  const wrongBookCount = await prisma.examWrongBook.count({
    where: { userId }
  });

  // 4. 按领域统计
  const qIds = [...new Set(allAnswers.map(a => a.questionId))];
  const questions = await prisma.examQuestion.findMany({
    where: { id: { in: qIds } }
  });
  const qDomainMap = new Map(questions.map(q => [q.id, q.domain]));

  // 聚合领域 → 答题数/正确数/正确率
  const domainStats = [...domainMap.entries()].map(([domain, s]) => ({
    domain,
    answered: s.answered,
    correct: s.correct,
    accuracyRate: s.answered > 0
      ? Math.round((s.correct / s.answered) * 1000) / 10
      : 0,
  }));

  return { totalAnswered, totalCorrect, accuracyRate, wrongBookCount, domainStats };
}
```

## 十、最佳实践与踩坑记录

### 10.1 最佳实践

#### 数据层

1. **种子题目全部用 `q()` 工厂函数**，确保字段完整性
2. **选项统一用 JSON 对象格式** `{"A":"内容"}`，前端渲染友好
3. **种子脚本必须是幂等的**（`findThenUpdateOrCreate` 模式）
4. **每道题必须关联 nodeId**，这是统计分析的基础维度
5. **`explanation` 字段必须写解析**，不只是答案

#### 渲染层

1. 所有题目字段（title/answer/explanation/options）都要过 `renderFormula()`
2. 答案字段 `correctAnswer` 在前端要用 `dangerouslySetInnerHTML` 渲染

#### 生成层

1. 带编号 `(1)...(2)...` 的枚举答案不适合出选择题 — 在生成器中过滤
2. 非数字答案（如文字描述、表达式）的干扰项生成需要更智能的逻辑
3. 判断题只翻转关键数字，不直接加"不是"——保持语义合理

### 10.2 已修复的 Bug

| Bug | 文件 | 根因 | 修复 |
|-----|------|------|------|
| 数字提取匹配到枚举题号 | `bulk-questions.ts` | `(1)3.14` 的 `1` 被当答案数字 | `stripPrefix()` + 长度守卫 |
| `seedModelQuestions` 非幂等 | `model-questions.ts` | 已存在只 skip | 改为 `updateOrCreate` |

### 10.3 当前系统限制

| 限制 | 影响 | 改进方向 |
|------|------|---------|
| 答案严格字符串匹配 | 填空 `$\\sqrt{2}$` ≠ `1.414` | 数学表达式化简匹配 |
| 干扰项生成简单 | 只有数字偏移，不产生逻辑干扰 | 利用内容 AI 生成 |
| 组卷纯随机洗牌 | 不保证难度/题型分布 | 加权分层抽取 |
| 无 frequency 控制 | 同一题可能反复出现 | 基于答题历史去重 |
| 模型训练不带关键词打分 | 关联节点题目质量不保证 | 按 tags 匹配度排序 |
