## 一、项目概述

### 愿景

打造一个**图文并茂、环环相扣**的全学科知识学习平台。让每个知识点不再是孤立的小点，而是一张可以追溯、可以展开、可以互动的知识网。

### 核心理念

> **"一个数乘以 1 表示正向，乘以 -1 表示反向，乘以虚数 i 表示旋转 90 度。"**

用直观的表达 + 可交互的方式，把抽象概念变成可感知的图形。每个知识点回答三个问题：
1. **这是什么？**（定义 + 图示）
2. **从哪来？**（前置知识，追溯根源）
3. **到哪去？**（后继知识，学了能做什么）

### 核心特性

| 特性 | 说明 |
|------|------|
| 🔗 **知识连续性** | 每个知识点有前驱后继，形成知识图谱 |
| 🎨 **图文并茂** | Canvas/Three.js 交互式演示 |
| 📊 **分层递进** | 按学科·领域层层递进 |
| 🧪 **训练+考试** | 知识点练习 + 自动组卷 + 错题本 |
| 🌐 **多端** | Web(主) + Tauri 桌面壳 |
| 🤖 **AI 辅助** | 知识问答、错题解析、学习推荐 |

## 三、技术栈

### 实际技术栈（非规划，是当前运行的）

```
前端:  React 18 + TypeScript + Vite 5 + Ant Design 5
       D3.js (力导向图谱) + Three.js (3D 几何)
       KaTeX (公式渲染) + Canvas 2D (互动演示)

后端:  Express 4 + TypeScript + Prisma ORM
       数据库: SQLite (dev) / MySQL (prod)
       JWT 认证 + bcryptjs 密码加密

AI:    Ollama (qwen2.5:3b) / OpenAI 兼容 API 接口
       Milvus 向量数据库 (RAG 知识问答)

桌面:  Tauri v2 (Rust), Node.js 子进程模式
       打包: .dmg (macOS) ~18MB

向量库: Milvus (知识检索 RAG)
```

### 为什么选 Node.js 而不是 Spring Boot？

项目最初规划了 Spring Boot 后端（见 `server/` 目录），但实际开发中换成了 Node.js。原因：
- 与前端共享 TypeScript 类型定义
- 原型迭代速度快
- Tauri 桌面端后端天然是 Node.js
- 使用 Prisma ORM 比 MyBatis-Plus 更简洁

> `server/` 目录（Spring Boot 骨架）是遗留代码，实际运行的是 `node-server/`。

## 五、知识体系设计

### 5.1 知识图谱数据结构

每个知识点是一个节点，边表示前驱/后继关系：

```
知识点节点 = {
  id: "MATH-01-001",         // 编码：学科-领域-序号
  title: "虚数单位 i",
  domain: "数",              // 所属领域
  level: "初中/高中",         // 难度层级
  difficulty: 1-5,           // 难度
  visualType: "canvas|model|static",
  summary: "一句话精髓",
  contentJson: {             // 结构化内容（JSON）
    text: "...",
    sections: [
      { type: "definition", title, content, formulas },
      { type: "keypoints", items },
      { type: "example", question, steps, answer },
      { type: "strategy", items },         // 解题策略
      { type: "common-mistakes", items },  // 易错点
      { type: "model-principle", ... },    // 模型原理
      { type: "recognition", ... },        // 识别方法
      { type: "derivation", content },     // 推导证明
      { type: "analogy", content },        // 类比理解
      { type: "related-formulas", items }, // 相关公式
    ]
  }
}

知识点关系 = {
  fromNodeId, toNodeId,          // 前驱 → 后继
  relationType: "prerequisite | next | reference | summary_of",
  description: "关系描述"
}

知识编码规则：{学科}-{领域编号}-{序号}
  数学: MATH-01-001
  物理: PHY-01-001
  化学: CHEM-01-001
  生物: BIO-01-001
  英语: ENG-01-001
  历史: HIST-01-001
  地理: GEOG-01-001
  政治: POL-01-001
  计算机: CMP-01-001
```

## 六、核心数据库设计（Prisma）

```
Node (知识点)
  nodeId       String @id      ← "MATH-01-001"
  title        String
  domain       String           ← 学科·领域 "数"、"力学"
  subject      String           ← 学科 "数学"、"物理"
  level        String           ← "初中"/"高中"
  difficulty   Int
  visualType   String?          ← "canvas"/"model"/"static"/null
  milestoneType String?
  summary      String?
  contentJson  String?          ← JSON
  sortOrder    Int
  status       Int

Relation (关系)
  id            Int @id @default(autoincrement())
  fromNodeId    String
  toNodeId      String
  relationType  String           ← "prerequisite"/"next"/"reference"/"summary_of"
  description   String?
  sortOrder     Int

ExamQuestion (题目)
  id            Int @id @default(autoincrement())
  nodeId        String
  title         String           ← 题干
  questionType  String           ← "choice"/"fill"/"judge"
  options       String?          ← JSON 选项
  correctAnswer String
  analysis      String?          ← 解析
  difficulty    Int
  tags          String?          ← JSON 标签
  status        Int

ExamPaper (试卷) / ExamAnswer (答题) / ExamWrongBook (错题本)
  — 标准考试系统设计
```

## 七、核心功能

### 7.1 知识图谱（D3.js 力导向图）

- 按学科/领域过滤，颜色编码
- 节点大小 = 难度，形状 = 普通/总结
- 边：实线=前置依赖，虚线=后继，点线=引用
- 交互：拖拽/缩放/悬浮高亮/点击跳转详情

### 7.2 Canvas 互动演示（13 个组件）

| 组件 | 知识点 | 学科 |
|------|--------|------|
| NumberLine | 正负数/数轴 | 📐 |
| ComplexPlane | 复数平面旋转 | 📐 |
| FunctionTransform | 函数变换参数调节 | 📐 |
| GeometryVisual | 勾股定理面积证明 | 📐 |
| TrigCircle | 单位圆三角函数 | 📐 |
| SequenceVisual | 等差/等比数列 | 📐 |
| CirclePower | 圆幂定理交互 | 📐 |
| CircularMotion | 圆周运动 | ⚡ |
| ForceDiagram | 受力分析 | ⚡ |
| OpticsDemo | 折射/反射 | ⚡ |
| MoleculeView | 分子结构 | 🧪 |
| ProbabilitySim | 概率模拟 | 📐 |
| PathOptimization | 最短路径规划 | 📐 |

### 7.3 逐帧动画播放器（7 个动画）

- 数的扩张 / 函数平移 / 方程求解 / 自由落体 / 复数旋转 / 方程配平 / 将军饮马

### 7.4 训练考试系统

- **题型支持**：选择题 / 填空题 / 判断题
- **练习模式**：随堂练 / 模型训练 / 章节练
- **考试模式**：自动组卷（按领域+难度）/ 提交批改 / 成绩统计
- **错题本**：自动收录 / 分类复习 / 移除
- **学习统计**：总正确率 / 各领域细分 / 薄弱点分析

### 7.5 AI 问答（Ollama / OpenAI 兼容）

- **知识点问答**：详情页底部 AI 导师面板
- **错题解析**：练习结果页「AI 深度解析」
- **学习推荐**：基于薄弱领域生成针对性练习
- **降级逻辑**：AI 不可用时自动返回规则化建议

### 7.6 LaTeX 渲染管线

全面覆盖的 KaTeX 渲染 + 6 层自动检测后备。详见 `docs/LaTeX-COMPLETE-GUIDE.md`。

## 八、启动与部署

### 开发模式

```bash
# 1. 后端
cd node-server && npx tsx src/index.ts

# 2. 前端
cd web && npm run dev

# 3. 桌面开发（可选）
cd web && npm run tauri:dev
```

### 桌面端打包（生产）

```bash
bash scripts/build-node.sh
```

产物：`src-tauri/target/release/bundle/dmg/知识动力_0.1.0_aarch64.dmg`（~18MB）

打包流程：
1. `[1/3]` Vite 构建前端 → `web/dist/`（~2.9MB）
2. `[2/3]` esbuild 打包后端 + 最小化 node_modules → `src-tauri/backend/`（~34MB）
3. `[3/3]` cargo tauri build → .dmg
